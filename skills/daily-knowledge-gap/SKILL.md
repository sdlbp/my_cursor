# Daily Knowledge Gap

## 快速说明
目标：从对话内容提炼知识盲点，按主题分别生成权威、专业的学习文档，并保存到 `~/Document/daily/YYYY/MM/DD/`。

## 触发条件
- 用户要求总结知识盲点、每日学习记录、学习薄弱点。
- 每轮对话结束时自动整理并落盘。

## 工作流
1. **识别盲点**：从对话中归纳不熟悉、模糊、易错或需补充的知识点。
2. **分主题建档**：每个主题单独创建一个文件，文件名根据薄弱点合理命名。
3. **目录结构**：按 `年/月/日` 生成目录，路径固定为 `~/Document/daily/YYYY/MM/DD/`。
4. **内容专业**：使用权威表述，必要时进行 WebSearch 获取可信来源。
5. **关联推荐**：在同目录或历史主题中查找相关内容，给出合理学习建议与关联阅读链接。

## 文档要求
- **格式**：Markdown (`.md`)
- **命名**：使用能清晰表达薄弱点的名称，例如 `sql-index-basics.md` 或 `分布式一致性-raft.md`
- **内容**：专业、权威、结构清晰；必要时引用可靠来源（给出链接）
- **关联**：可在“关联阅读/延伸学习”中引用其他主题文件或外部参考

## 内容模板
使用以下模板生成每个主题文件：

```markdown
# [主题名称]

## 概览
[用 3-6 句概括该主题的核心概念与边界]

## 关键概念
- [概念 1：权威定义 + 简短说明]
- [概念 2：权威定义 + 简短说明]

## 常见误区
- [误区 1：描述 + 正确理解]
- [误区 2：描述 + 正确理解]

## 学习路径
1. [基础前置知识]
2. [核心知识]
3. [进阶方向]

## 关联阅读
- [本地主题文件：相对路径或文件名]
- [外部参考：权威链接]

## 练习建议
- [实践 1]
- [实践 2]
```

## 质量要求
- 语言正式、严谨，避免口语化。
- 若引用外部资料，优先选择官方文档、标准规范、权威教材或高质量技术文章。
- 若对话信息不足，允许补充必要背景，但需保持可验证性。
