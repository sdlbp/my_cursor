# Daily Knowledge Gap

## 快速说明
目标：从对话内容提炼知识盲点，按主题分别生成权威、专业的学习文档。落盘路径为工程根下 `.daily`（符号链接指向 `~/Documents/daily/`）。

## 触发条件
- **仅主动调用**：用户明确要求总结知识盲点、记录每日学习或生成盲点文档时执行。


## 工作流
1. **挂载当日目录**：在工程根目录下，确保 `.daily` 为指向 `~/Documents/daily` 的符号链接。
   - **检查**：若已存在 `.daily` 且为指向 `~/Documents/daily`（或 `~/Documents/daily/`）的符号链接（可用 `readlink .daily` 或 `ls -l .daily` 判断），则直接使用，不重复创建。
   - **否则**：先执行 `mkdir -p ~/Documents/daily/docs/YYYY/MM/DD/`（取当日 `YYYY/MM/DD`），再在工程根执行 `ln -sfn ~/Documents/daily .daily`。
2. **识别盲点**：从对话中归纳不熟悉、模糊、易错或需补充的知识点。
3. **分主题建档**：每个主题在 `.daily` 下单独建一个文件，文件名根据薄弱点合理命名。
4. **内容专业**：使用权威表述，必要时进行 WebSearch 获取可信来源。
5. **关联推荐**：在 `.daily` 或 `~/Documents/daily/docs/` 下的历史主题中查找相关内容，给出合理学习建议与关联阅读链接。

## 文档要求
- **格式**：Markdown (`.md`)
- **命名**：使用能清晰表达薄弱点的名称，例如 `sql-index-basics.md` 或 `分布式一致性-raft.md`
- **内容**：专业、权威、结构清晰；必要时引用可靠来源（给出链接）
- **关联**：可在“关联阅读/延伸学习”中引用其他主题文件或外部参考

## 内容模板
使用以下模板生成每个主题文件：

```markdown
# [主题名称]

## 概览
[用 3-6 句概括该主题的核心概念与边界]

## 关键概念
- [概念 1：权威定义 + 简短说明]
- [概念 2：权威定义 + 简短说明]

## 常见误区
- [误区 1：描述 + 正确理解]
- [误区 2：描述 + 正确理解]

## 学习路径
1. [基础前置知识]
2. [核心知识]
3. [进阶方向]

## 关联阅读
- [本地主题文件：相对路径或文件名]
- [外部参考：权威链接]

## 练习建议
- [实践 1]
- [实践 2]
```

## 质量要求
- 语言正式、严谨，避免口语化。
- 若引用外部资料，优先选择官方文档、标准规范、权威教材或高质量技术文章。
- 若对话信息不足，允许补充必要背景，但需保持可验证性。
